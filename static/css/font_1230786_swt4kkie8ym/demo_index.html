Unicode 引用

Unicode It is the most primitive way of using fonts on the web. Its characteristics are:

  • It has the best compatibility, supports IE6+, and all modern browsers.
  • Supports dynamic adjustment of icon size, color, etc. by font.
  • But because it is a font, multi-color is not supported. Only single-color icons in the platform can be used. Even if there are multi-color icons in the project, it will be automatically decolorized.

Note: The new version of iconfont supports multi-color icons. These multi-color icons cannot be used in Unicode mode. If there is a need, it is recommended to use the symbol reference method

Unicode The use steps are as follows:

The first step: copy the generated under the project @font-face

@font-face {
  font-family: 'iconfont';
  src: url('iconfont.eot');
  src: url('iconfont.eot?#iefix') format('embedded-opentype'),
      url('iconfont.woff2') format('woff2'),
      url('iconfont.woff') format('woff'),
      url('iconfont.ttf') format('truetype'),
      url('iconfont.svg#iconfont') format('svg');
}

Step 2: Define the style using iconfont

.iconfont {
  font-family: "iconfont" !important;
  font-size: 16px;
  font-style: normal;
  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;
}

Step 3: Select the corresponding icon and get the font code, apply it to the page

<span class="iconfont">&#x33;</span>

"iconfont" It is the font-family under your project. It can be viewed by editing the project, the default is "iconfont".

font-class Quote

font-class It is a variant of the way Unicode is used, mainly to solve the problem of unintuitive writing and unclear semantics in Unicode.

与 Unicode Compared with the way of use, it has the following characteristics:

  • Good compatibility, support IE8+, and all modern browsers.
  • Compared with Unicode, the semantics are clear, and the writing is more intuitive. It is easy to distinguish what this icon is.
  • Because the class is used to define the icon, when you want to replace the icon, you only need to modify the Unicode reference in the class.
  • However, because the font is essentially used, multi-color icons are still not supported.

The use steps are as follows:

Step 1: Import the fontclass code generated under the project:

<link rel="stylesheet" href="./iconfont.css">

Step 2: Select the corresponding icon and get the class name to apply to the page:

<span class="iconfont icon-xxx"></span>

" iconfont" It is the font-family under your project. It can be viewed by editing the project, the default is "iconfont".

Symbol Quote

This is a brand new way to use it, and it should be said that this is the mainstream in the future, and it is also the recommended usage of the platform. Related introduction can refer to thisarticle This usage is actually a collection of SVG, which has the following characteristics compared with the other two:

  • Multi-color icons are supported, and are no longer limited by monochrome.
  • Through some skills, support like fonts, through font-size, color To adjust the style.
  • Compatibility is poor, support IE9+, and modern browsers.
  • The performance of browser rendering SVG is average, not as good as png.

The use steps are as follows:

Step 1: Import the symbol code generated under the project:

<script src="./iconfont.js"></script>

Step 2: Add general CSS code (introduce it once):

<style>
.icon {
  width: 1em;
  height: 1em;
  vertical-align: -0.15em;
  fill: currentColor;
  overflow: hidden;
}
</style>

Step 3: Select the corresponding icon and get the class name, and apply it to the page:

<svg class="icon" aria-hidden="true">
  <use xlink:href="#icon-xxx"></use>
</svg>
